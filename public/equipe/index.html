Organização do Trabalho#

  • Modelo ágil utilizado (Scrum, XP etc.):

    • Scrum.

  • Sprint planning, reviews e retrospectivas:

    • Sprint 0: Objetivos:
      • Entender conceitos de requisitos de software.
      • Estudar metodologias ágeis.
      • Estudar Git/Github.
    • Sprint 1: Objetivos:
      • Nos familiarizarmos com as ferramentas que estariamos utilizando, como o figma.
      • Analisar os tipos de arquiteturas que poderíamos utilizar no projeto.
      • Estudar a respeito de API’s e como implementá-las.
    • Sprint 2: Objetivo:
      • O foco foi levantar os requisitos e escopo iniciais do projeto.
      • Também ocorreram avanços na documentação do projeto, como a implementação inicial do README e github pages.
      • Foram estudados tópicos relevantes para o projeto, como os frameworks fastAPI e React, protocolo HTTP, benchmark e arquitetura cliente-servidor.
    • Sprint 3: Objetivo:
      • O foco foi em desenvolver a documentação necessária para a R1, com o deploy das github pages e início em definitivo do protótipo pelo figma.
      • Também seguimos com estudos sobre os frameworks React e fastAPI e sobre testes de software.
      • Fizemos análises à respeito do banco de dados e arquitetura que serão utilizados.
    • Sprint 4: Objetivo:
      • O foco foi em progredir a documentação e o código funcional referente à release 1.
      • Estudamos tópicos como fastAPI, webscraping usando Scrapy e React.
      • Pesquisamos também maneiras de consumir APIs que possam contribuir no projeto.
    • Sprint 5: Objetivo:
      • Focamos em refinar algumas implementações que fizemos na R1, como adicionar efetivamente a rota /sobre e implementar mudanças em relação ao frontend.
      • A equipe priorizou também em seguir estudos relacionados a prática de webscraping, como implementar spiders em portais de notícia e no estudo de React.
    • Sprint 6: Objetivos:
      • Focamos em implementar funcionalidades novas ao projeto.
      • Implementamos os cards de notícia ao frontend.
      • Demos início ao banco de dados em MongoDB.
      • Fizemos uma implementação inicial das spiders no portal de notícia do g1 para webscraping.
      • Conseguimos treinar a api do gemini para validação de notícias coletadas.
    • Sprint 7: Objetivo:
      • Nesta semana, o foco foi o desenvolvimento do banco de dados como algo já funcional.
      • Focamos na conexão do banco de dados em MongoDB com o backend com fastapi.
    • Sprint 8: Objetivos:
      • Implementação do mecanismos principal de coleta de notícias: os web crawlers.
      • Coleta dos dados no modelo estabelecido pelos integrantes de banco de dados.
      • Estabelecer conexões do backend com o frontend, mostrando já notícias presentes no banco de dados.
      • Aprimoramento da rota “Sobre” no frontend.
    • Sprint 9: Objetivos:
      • Nesta sprint, foi dado início às implementações de testes unitários, começando pelo backend.
      • Inserimos mais spiders à parte de webscraping do projeto.
      • Foi automatizado o armazenamento de notícias do webscraping para o banco de dados.
      • Foi feita a comunicação do banco de dados com o backend, e repassamos essas informações para o front end.
      • Além disso, houveram modificações no frontend, como o início do design responsivo da página, desenvolvimento dos filtros .de pesquisa e desenvolvimento do pop-up de detalhamento de cada card de notícia.
    • Sprint 10: Objetivos:
      • Nesta sprint, o time focou em desenvolver mais a fundo o frontend.
      • Obtivemos progresso na responsividade para outros tipos de tela.
      • Desenvolvemos o pop up de detalhamento de cada notícia de acordo com o protótipo proposto no figma.
      • Foram feitos testes dos principais componentes da pagina de notícias.
      • Notamos falhas na segurança do projeto que permitiam que o usuário tivesse acesso a rotas não permitidas, mas que já foram devidamente resolvidas.
    • Sprint 11: Objetivos:
      • Na sprint 11, o time centralizou o foco na preparação para a reta final do projeto.
      • Dentre as tasks que fizemos, está o início da dockerização do projeto, para posteriormente facilitar o deploy.
      • Algumas mudanças no frontend aconteceram, como a implementação de botões de filtro, que caracterizam a notíca entre “fake”, “duvidosa” ou “verdadeira”.
      • Também houveram mudanças no figma para que consigamos entender melhor o que é desejado para a parte visual do projeto, além de deixa-lo mais apresentável para a release.
      • Por fim, foi implementada a coleta automática das notícias com as spiders criadas.
    • Sprint 12: Objetivos:
      • Nesta sprint foram finalizados a maior parte dos testes unitários do projeto, incluindo agora testes para as spiders do web scraper, a pagina sobre do frontend e uma refatoração das funções de teste do backend.
      • Além disso, foi feito um workflow de CI para o github actions que valida modificações no backend, frontend e web scraper baseado nos testes implementados.
    • Sprint 13: Objetivos:
      • Na nossa sprint final para a Release 2, focamos em refinar as features já implementadas no projeto.
      • Dentre as melhorias realizadas, foi resolvido um bug em relação à página de notícias, em que a ordem de disposição das notícias estava invertida, assim como alguns erros de comunicação com o backend.
      • Foram implementadas, também, a spider da Jovem pan para coleta de notícias pelo web scraper.
      • A rota de /painel foi desenvolvida, em que é apresentado estatísticas sobre as notícias coletadas.
      • Houve, também, a implementação de Dockers para facilitar o deploy do projeto.
      • Foi desenvolvido um verificador de notícias duplicadas utilizando a LLM.
      • Adicionalmente, foram melhorados as rotinas de CI/CD com workflows e a implementação de linters para verificação e teste de pushs e pull-requests.

  • Ferramentas adotadas:

    • Git/Github: para versionamento de código e desenvolvimento de workflows(CI/CD).
    • Figma: para organização do projeto e desenvolvimento de dashboards.
    • Hugo: para criar a gitpages dentro do github.
